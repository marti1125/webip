1.                Acerca del Manual

 

 

Este Manual de Usuario contiene  instrucciones sobre cómo utilizar SAFECITY Stream Server versión 2.0.  Cubrirá los aspectos básicos desde inicializar el servidorhasta algunos temas  avanzados como modificar archivos de configuración.

 

Este Manual está dirigido a usuarios de SAFECITY Stream Server.  Se espera que el lector tenga los conocimientos básicos de multimedia streaming y las tecnologías requeridas para realizar streaming.

 

 

 

2.                AcercadeSAFECITY Stream Server (SCSS)

 

 

2.1.             ¿Qué es el SCSS?

 

SAFECITY Stream Server es una Aplicación Servidor capaz de entregar contenidos de video en directo y/o bajo demanda a cualquier dispositivo agran calidad.  Con SCSS puedetransmitir su audio/videoa todas las plataformasmás populares incluyendo Adobe Flash, Apple, dispositivos iOSy Quick Time.

 

 

2.2.             ¿Porqué utilizar el SCSS?

 

La arquitectura única de SCSSaumenta significativamente el rendimiento de I/O en comparación con aplicaciones basadas en Java, y es la única aplicación unificada capaz de funcionar prácticamente en cualquier plataforma (Linux, Windows, Mac OSX, etc), incluyendo dispositivos integrados (codificadores, cámaras IP, grabadoras de DVD y otros).

 

 

2.3.        Características y Beneficios delSCSS

 

SAFECITY Stream Server no es solo una aplicación multi-formato y multi-protocolo que transmite contenido a través de diferentes pantallas y plataformas.  SCSS es la aplicación de transmisión más eficiente y flexible. Ofrece gran capacidad de transmisión a un costo de funcionamiento reducido.  Para mayor referencia, ver el siguiente gráfico.

 

 

 

Alta Eficiencia

El SCSSrequiere de un consumo pequeño de CPU, sin embargo es capaz de manejar aproximadamente 2,000 conexiones simultáneas por núcleo corede CPU.En otras palabras, nunca sobrepasará sus recursos de hardware antes de llegar al límite de su ancho de banda.

 

Extensible

Nunca más genere módulos personalizados o esté limitado a un solo lenguaje de programación para extender funcionalidad a su aplicación e infraestructura. SCSStiene una serie de APlsdiversos, incluyendo standard HTTP calls, PHP, Lua, o C++, permitiendo una integración fácil y rápida del SCSS en flujos de trabajos existentes.

 

Unificado

Le permite ingresar unstream envivo H.264 de un codificador MPEG-TS,RTMP, oRTP y transformarlo simultáneamente y redistribuir el streama cualquier otro punto final incluyendo PCs, Macs, teléfonos móviles, tablets y televisores. Nuestro compromiso con los estándares le asegura que SCSS cumple a cabalidad con cada protocolo indicado.

 

Cross Platform

Distintas plataformas diseñadas desde cero para ser realmente una plataforma compatible capaz de operar en virtualmente cualquier sistema operativo incluyendo codificadores, cámaras IP, DVRs, y mucho más.

 

Escalable

Ya sea funcionando con pocos usuarios o con cientos, SCSScumple con sus requerimientos de  transmisión en vivo o bajo demanda, a través de unsólido equilibrio de carga que le permite escalar ilimitadamente según sus requerimientos, mientras los costos de hardware y licenciamiento se mantienen al mínimo.

 

Confiable

El SCSSestá probado bajo condiciones de alto tráfico y los más altos requerimientos que exigen máxima disponibilidad y fiabilidad.

 

 

2.4.             ¿Cómofunciona el SCSS?

 

El SCSScorre como una aplicación independiente a la que puede enviar transmisiones de video y audio. Puede conectarse con el SCSSmediante una variedad de programas u otros servidores y utilizar el API para enviar transmisiones o para ingresar nuevosstreams.

 

 

 

2.4.1.       Direccionamiento de Transmisiones

 

 

El variado conjunto de APIs del SCSS incluye Envío e Ingreso de streaming, el cual le permite fácilmente publicar o recibir transmisiones RTMP/RTSP/HLS/MPEG-TS/otroshacia o desde otros lugares, como un CDN o un proveedor de servicios.

 

 

2.4.2.       Transformación de Stream

Ya sea que desee publicar RTMP desde RTSP, HLS desde MPEG-TS, o cualquier otra combinación posible de los protocolos de streaming, SCSSunifica todas las tecnologías de streaming en un solo flujo de trabajo.

 

 

 

 

2.5.             ¿Dónde va a funcionar?

 

Prácticamente en todo.Se ejecuta en Windows, Linux, Mac OSX, BSD y Solaris.

Específicamente, el SCSSpuede correr en:

Windows 7, Vista, Server 2008*

Debian Linux

CentOS/RedHat Linux

Ubuntu Linux

SUSE Linux

Mac OSX

FreeBSD

*Nota:  El SCSS no corre en Windows XP, Windows Server 2003 o anterior.

 

 

2.6.             ¿Con qué se puede conectar?

 

El SCSSse puede conectara cualquier dispositivo que envíe un stream estándar. Al SCSSse puede ingresar RTMP, RTSP/RTP, MPEG-TS, LiveFLV. El SCSS también se puede configurar para recibir una transmisión directamente de un chip codificador de hardware (para aplicaciones integradas).

 

 

 

3.                Instalación eInicio

 

3.1.        Contenidos de Distribución

 

La distribución del SAFECITY Stream Server incluye las siguientes carpetas:

 

SafeCity/

En Windows, este directorio contiene los siguientes archivos. Para todos los demás sistemas operativos esta carpeta no contiene archivos, sólo los subdirectorios que se indican a continuación.

 

Run.bat Usar para ejecutar SCSScomo una aplicación de consola. Esto es lo ideal para las pruebas iniciales, depuración e integración, ya que muestra el registro de errores en la consola.

 

WinService-Create.bat - Crea un servicio de Windows para el SCSS, a fin de que se pueda ejecutar como un proceso de fondo.  Esto DEBE ser ejecutado por un Administrador.Si tiene permiso de Administrador, puede hacer click derecho y seleccionar "Ejecutar como Administrador". Esto CREARÁuna entrada en el registro de su sistema.

 

WinService-Remove.bat  Detiene el Servicio de Windows del SCSSy elimina el servicio del registro. Esto DEBE ser ejecutado por un Administrador. 

 

WinService-Start.bat  Inicia el Servicio de Windows del SCSS(Inicia el servidor SCSS).

 

WinService-Stop.bat  Detiene el Servicio de Windows del SCSS(Detiene el servidorSCSS).

 

Evostreamms.exe Es la Aplicación SCSSpropiamente.

 

Srvany.exe  Este es un binario proporcionado por Microsoft y se utiliza para crear el Servicio de Windows.

 

 

 

 

SafeCity/config

 

Esta carpeta contiene todos los archivos de configuración utilizados por SCSS.

 

Config.lua  El archivo de configuración principal utilizada por elSCSS. Los contenidos de este archivo se detallan en este Manual más adelante.

 

License.lic _El archivo de licencia (para sistemas Windows).

 

Users.lua  Define la autenticación válida que el servidor requerirá cuando los streamssean enviados al SCSS.

 

pushPullSetup.xml  Este archivo es utilizado por el SCSSpara almacenar comandos de acción de stream que se realizan a través del API.Este archivo es SOLO PARA USO INTERNO DE SCSS. EDITAR MANUALMENTE ESTE ARCHVO ES PELIGROSO Y PUEDE CAUSAR EFECTOS INDEFINIDOS.  Debe interactuar con el SCSSúnicamente a través de las funciones API.

 

connlimits.xml  Define el número máximo de conexiones que desea que el SCSSacepte.

 

 

SafeCity/doc/

 

El directorio doc contiene la documentación del SCSS.

 

API Definition.pdf  Descripciones de todos los APIs del SCSS.

 

EMS How Tos.pdf  Brinda comandos de ejemplo para realizar tareas básicas con el SCSS.

 

 

SafeCity/media/

 

Este directorio es la ubicación por defecto para los archivos de video bajo demanda. Aquí es donde el SCSS buscará cuando se realicen solicitudes de video. Esta ubicación predeterminada se puede cambiar en el archivo de configuración principal del SCSS, que es típicamente config/config.lua

 

 

SafeCity/logs/

 

Este es el directorio donde el SCSSgrabará sus registros.  Esta ubicación predeterminada se puede cambiar en el archivo de configuración principal del SCSS, que es típicamente config/config.lua

 

 

SafeCity/web_ui/

 

Proporciona una interfaseweb para usuario que se puede utilizar para administrar el SCSS. Esta interfase web tendrá que ser instalada en un servidor web (como Apache o MS IIS) que está habilitado con PHP. Esta carpeta también contiene PHP y JavaScript wrappers para el API del SCSS. Ver el APIDefinition.pdf para mayor información sobre el API.

3.3.        Definición de Línea de Comandodel SCSS

 

El ejecutable evostreamms se puede ejecutar con algunas opciones diferentes. El modelo de la línea de comandos es la siguiente:

 

evostreamms[OPCIONES] [ruta de archivo de configuración]

 

 

OPCIONES

 

--help

Muestra la ayuda y sale.

 

-- version

Muestra la versión y sale.

 

--use-implicit-console-appender

Añade registro extra en la ejecución, pero solo es efectivo cuando se inicia el servidor como aplicación de consola. Esto es particularmente útil cuando el servidor se inicia y se detiene inmediatamente por razón desconocida.  Le permitirá ver si algo está mal, sobre todo con el archivo de configuración.

 

--daemon

Sobrepasa la configuración daemon dentro del archivo de configuración y fuerza al servidor a iniciar en modo daemon.

 

--uid=<uid>

Ejecutar el proceso con el ID de usuario especificado.

 

--gid=<gid>

Ejecuta el proceso con el ID de grupo especificado.

 

--pid=<pid_file>

Crear un archivo PID.  Solo funciona si se especifica la opción --daemon.

Cuando los streams son seleccionados de la Lista de Streams, el SCSSmostrará los URLsque se pueden utilizar para obtenerlos del sistema.  El URL RTMP puede ser utilizado por reproductores de video basados ​​en Flash. 4.4.            Crear HLS (HTTP Live Streaming)

 

El SCSSpuede crear un stream HLS de su stream origen para que pueda ser visto por dispositivos Apple iOS (iPhone, iPad, etc). Utilice la opción "Crear un stream HLS" para crear la transmisión HLS:

 

 

Los parámetros que seleccione son importantes, así que sea cuidadoso con lo que está usando.

 

La Carpeta Destino debe ser la "ruta web", o la carpeta pública, de un servidor web. Probablemente sea mejor utilizar el mismo destino donde instaló la IU.

 

El Nombre de Grupo se utiliza para organizar su stream HLS. Si tiene varias copias del mismo stream (usualmente en diferentes bit-rates resoluciones), debe usar el mismo Nombre de Grupo. En caso contrario, es importante que utilice diferentes Nombres de Grupos para diferentes streams.

 

El parámetro de ChunkLengthdefine cuán grande (en segundos) es los chunk de archivos HLS. Es mejor dejar esto en 10, a menos que esté seguro de lo que está haciendo.  Indicar un número demasiado pequeño causará que los dispositivos iOSesténconstantemente descargando  archivos y esto afectará severamente el funcionamiento.
